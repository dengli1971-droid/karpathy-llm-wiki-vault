# Claude 4.8发布：市场好评不断 / 禁入条款放开、Codex给的压力不小 / 算力支撑到位，ARR年底预期有望达成

**日期**: 2026-06-06 | **来源**: acecamptech.com | **公司**: AWS, Amazon, Anthropic, Broadcom, DeepSeek, Google, NVIDIA, 千问 | **标签**: AI推理, 大模型, 边缘推理, 消费 | **Ticker**: AMZN US, AVGO US, GOOGL US, NVDA US

## 核心观点
- Claude模型迭代性能，对得起版本更新。这也就是用头部厂商模型的优势，基本不用担心迭代完后的性能校验。身边用户在体验完之后，基本完全切换到最新的模型了
- 随着Codex近期数据的高歌猛进，Anthropic联合多家厂商解决了算力之后，终于是放开了禁入；一方面有足够的算力提供稳定、可靠、挣钱的服务；另一方面Codex的数据增长太好了，作为竞争对手也需要出台和更新策略来进行打压。对于Anthropic自身而言，更好用户体验的/fast模式依旧提供，用户选择性增强
- 结合上述谈到的背景，无论是模型迭代+算力问题解决+放开禁入（套餐宽松条件）等市场策略的进行，本质上依然是2026年以及未来的ARR作准备，市场预期存在一直高涨的可能，主要是Anthropic的数据表现太亮眼

## 正文

# 1、背景

**2026年5月底，Anthropic推出了最新一代模型Claude 4.8，较上一代Claude 4.7的推出，也仅仅过去了一个多月的时间，也恰好和谷歌推出Gemini 3.5 flash在同一个月份。当下市场，各大头部厂商推出的模型速度越来越快，也逐渐在反应这个赛道的竞争加剧。并且，诸如Open AI虽并未推出最新模型，但是外网也通过Codex看到了其GPT-5.6的模型泄漏，以及近期其一直在加大对于Codex的迭代，工程应用在短时间里的版本迭代也被提速了。综上所述，激烈的市场竞争+格局未定的时刻+市场潜在价值急需挖掘和变现的机会，迫使各大厂商更加专注在了Anthropic所一直延续到赛道，AI的发展也逐渐从几年前的发散模式回归到聚合模式，统一朝着焦点方向发力**

**当然，笔者认为这里只是一个中间态，也不乏其他方向的探索，只不过依然是市场需求驱动，即需求在哪里，模型厂商、咨询、关注焦点、流量就在哪里。好比之前的Sora模型时刻，只不过现在Anthropic所推崇的AI Coding编程、长任务和复杂模式有了具体的落地价值和意义。诸如谷歌在强调的多模态能力，以及端侧的优势，尽管当下对于用户的体感并不是特别明确、以及用户在这项技术的价值获得感有限；但是，量变实现质变，当下的拐点只是还未到来而已，可能是市场的认知问题、也可能是技术先进性问题，均需要时间来推动解决上述提到的不足，衍生出新的市场和新的焦点**

# 2、模型效果实测、价格分析以及市场接受度

**Claude 4.8自发布以后，热度不断。以及市场和周边用户的体验来看，效果非常不错，符合其公布的效果对比和测评。其中，较为核心的提升如下：**

**1、针对大模型 "证据不足时仓促下结论、高置信度输出未经验证内容" 的典型缺陷，Opus 4.8进行了根本性改进：**

- **不确定性表达能力增强：面对不确定信息时，更倾向于主动标记疑点、明确表达不确定性，而非 "强行给出答案"**

- **无依据结论倾向大幅下降：产生虚假或未验证内容的概率显著降低。对齐性已接近Anthropic目前表现最好的Claude Mythos预览版。所以，只要其内部有最优的模型，尽管未发布，但是已经表明技术达到了这种水平，后续的能力输出完全取决于公司的输出时间选择，其实有点像之前Open AI领先时，一直在把控全球AI的技术迭代节奏**

- **代码安全性提升：允许代码中存在缺陷却未加提示的概率，仅为前代 Opus 4.7 的四分之一**

**2、整体来看，Claude Opus 4.8在6项测试中拿下了5项第一，仅在终端编码能力上略逊于GPT-5.5，不仅全面超越了前代Opus 4.7，还大幅拉开了与GPT-5.5和Gemini 3.1 Pro的差距：**

- **在SWE-Bench Pro真实代码库智能体编码测试中，Opus 4.8 以 69.2% 的成绩领先 GPT-5.5 高达 10.6 个百分点，是其最核心的竞争力**

- **在 Terminal-Bench 2.1 终端智能体编码测试中，Opus 4.8 虽以 74.6% 不及 GPT-5.5 的 78.2%，但相比前代提升了 8.5 个百分点，与 GPT-5.5 的差距从 Opus 4.7 时代的 12.1 个百分点大幅收窄至 3.6 个。其实，这里的结果是比较惊奇的，属于测试和实际不太相符的情况。Claude在做类似的任务时，效果上明显优于GPT-5.5，特别是在比较难的问题解决上。所以，笔者也一直强调是参考的意义，借鉴的意义的较小，即成绩和现实情况存在冲突**

- **在 Humanity's Last Exam 跨学科推理测试中，Opus 4.8 无论有无工具都保持绝对领先，无工具模式 49.8% 领先 GPT-5.5 8.4 个百分点，有工具模式 57.9% 领先 5.7 个百分点**

- **在 OSWorld-Verified 智能体计算机操作测试中，Opus 4.8 以 83.4% 的成绩继续保持领先**

- **在衡量综合能力的 GDPval-AA 知识工作测试中，Opus 4.8 以 1890 分断层第一，领先 GPT-5.5 121 分，领先 Gemini 3.1 Pro 高达 576 分**

**从测评的结果来看，无论是对齐能力，还是测评的综合表现，Claude Opus 4.8已成为当前综合能力最强的模型，尤其在软件工程、综合知识工作等领域，而Gemini 3.1 Pro在所有核心智能体能力上均明显落后于前两者。这也是谷歌层面面对的危机，即市场表现热潮，但是自身模型的表现不及核心竞争对手。最后，尽管Claude Opus 4.8上线周边的使用体感和评价都非常好，对得起测评的分数。但是，市场也有一些小问题反馈：**

**1、一个简单的交互，花了0.2美元；即简单问题也会思考复杂，消耗大量的Token。虽然模型价格没有变化，但是这种消耗Token的思维链，也是折算了涨价的一部分内容**

**2、根据市场消息收集显示，在Claude Opus 4.8发布当日，即引发****蒸馏与双标争议****。有多位开发者通过裸API调用测试发现，模型会自称是千问大模型和DeepSeek；但是，该现象在官方客户端较难复现。推测的原因可能是与产品层更严格的系统提示词约束有关，或者是部分第三方API中继服务会将Anthropic格式请求自动转发至DeepSeek端点，测试者可能并未调用到真正的Claude。由于Anthropic此前曾公开指责国内公司对Claude输出进行蒸馏违反服务条款，此次自家模型出现身份混乱，迅速在社交媒体上引发 "双标" 质疑**

# 3、Codex压力以及Anthropic再次放开禁入，ARR突破的期望尽在眼前

## 3.1、更快模式发布 + 降价策略

**在此次的发布中，除了常规的Claude Opus 4.8的发布。Anthropic还带来了Fast Mode，一种很快的思考模式和更短的响应时间。用户可以在 Claude Code中，通过使用 /fast 打开 Fast Mode。打开后会出现一个闪电感的标识。当然，这种模式带来的弊端是需要支付更高的价格，即享受快速性的同时，支出独占的算力资源，对比于常规模式，Fast Mode的价格是常规模式的2倍；但是对比于之前的Claude Opus 4.7版本，其价格算是倍打下来了，仅有之前模型的相同模式下的1/3价格。详细列举如下：**

**模式 输入价格 (每百万 token) 输出价格 (每百万 token) 速度**

**Opus 4.8 标准模式 $5 $25 1x**

**Opus 4.8 Fast Mode $10 $50 2.5x**

**Opus 4.7 Fast Mode $30 $150 2.5x**

**这种模式针对于当下不差钱的公司或者说任务紧急的情况是一个福音，说白了就是时间+金钱换效率。以日常在150美元的消耗来看，这种模式下会到300美元的价格。另外，如果以月终来计算，全月的消费将达到6000美元左右，大多数企业难以负担。但是，也不乏鼓励全员的公司可以支撑到位，例如英伟达。笔者了解到的信息，其内部支撑的是2.5万美元/月的消耗，以及鼓励员工使用AI。并且，以笔者了解到的信息，大多数Manager也主动和鼓励team使用这项技术，从分享的内容了解到，不少Manager认为大多数内部员工，其实已经达不到Claude Opus 4.8这样的模型水平了，无论是全面性还是单项的专业能力。在拥抱AI的同时，各类问题的解决效率也被极大的提高，已经有Manager层面，从之前的个人Coding到现在，一个月基本无需在写代码了，并且效率、以及提交量提高了若干倍**

## 3.2、Anthropic放开禁入，Codex功不可没

**在2026年4月份时，Anthropic做了一个强硬动作，即禁止Claude订阅用户（Pro/Max等）将订阅额度用于第三方 Agent 工具（如 OpenClaw 等）。如果想继续用，必须走 API 单独购买 usage，额外付费。对于一般的客户而言，单独的计费模式，基本无法使用。例如，现在的模型的定价方式是：Opus 4.8 标准模式 $5 $25； 如果折中计算，百万token的消费会在15美元左右。假设每日的消耗在0.1B（算是一个较为常用用户的 单日消耗量），其每日的成本是：15 * （0.1B / 1M） = 1500 美元**

**当时Anthropic给的理由也很明确，其认为Agent使用方式与普通聊天完全不同，Agent 会连续调用模型、读取上下文、执行步骤、检查结果、反复迭代，消耗成本远高于普通用户。Anthropic当时完全是为了止血，即防止用户用固定月费无限薅高成本服务**

**但是，随着Codex的迭代加速，以及不少用户投入到Open AI的怀抱，以及Anthropic的算力逐渐在被解决。Anthropic也最终愿意开放这个渠道了，其宣布打开禁入，重新开放权，这项服务的名为Agent SDK credits：即2026年6月15日起，第三方 Agent、Claude Agent SDK、claude -p 等程序化可以使用这项服务了，但不再是无限使用，而是改为单独计量、额度制。当然，从规则来看，是开启了一种比较折中的方式。规则是：**

**从2026年6月15日开始，Claude Pro、Max、Team、Enterprise 用户，可以按不同套餐领取一笔专门用于 Agent SDK 的月度额度，其中Pro 用户是 20 美元、Max 5x 是 100 美元、Max 20x 是 200 美元；用完之后，依然按照API的计费模式。不过，按照市场的行为，资源池是一个很好的解决方式，通过这种模式，间接可以将所有的账号额度打包，起到完全放开的作用，只不过这是用户层面的操作，而不是Anthropic的主动行为**

**对于Anthropic而言，愿意这样去做的理由很明显：**

**1、Open AI Codex的增长过快，Anthropic必要收拢服务间隙，防止被Open AI钻空子：根据市场数据显示，Open AI Codex 在2026年上半年呈现爆发式增长，从2026年年初的约60万周活跃用户一路飙升至500万周活跃用户（2026年5月31日官方宣布），短短5个月内增长了8.3倍。特别的，在4月～5月份两个2月期间，增长周活跃突破200万用户，这里少不了Open AI全力以赴的迭代；但是，Anthropic的封禁转移用户，助力也不少：**

- **2026 年 4 月 8 日：达到300 万周活用户（Sam Altman 确认）**

- **2026 年 4 月 21 日：突破400 万周活开发者，仅两周就新增 100 万用户**

- **2026 年 5 月 31 日：正式突破500 万周活用户大关，产品负责人宣布全面重置所有用户使用限额**

**2、算力问题逐渐解决，强大的算力资源池也需要提到使用率，并且尽管推出套餐和用户可以使用间接的方式来薅羊毛。但是，对于Anthropic而言，套餐的方式依然有利可图，并且继续其IPO和用户维持而言、以及竞争对手的打击，开放禁入都是一种不错的方式。Anthropic近期获得了大量的算力支撑，详细列举如下：**

- **SpaceX：450亿美元锁定英伟达GPU算力，解决当下算力不足问题（2026年5月6日）**

- **合作规模：未来三年支付近 450 亿美元，每月 12.5 亿美元（5-6 月享受折扣费率）**

- **算力资源：独家使用 Colossus 1 数据中心全部算力（222,000 + 张 NVIDIA H100/H200/GB200 GPU，300 + 兆瓦电力），后续扩展至 Colossus 2 数据中心**

- **即时影响：合作宣布当日，Anthropic 立即取消了 Claude Pro/Max 用户的峰时限流，将 Claude Code 5 小时使用限额翻倍至 10 小时，并大幅提升了 Opus API 的速率限制**

- **亚马逊 AWS：十年千亿锁定自研芯片算力，解决未来算力渠道单一问题（2026年4月20日）**

- **合作规模：亚马逊立即投资 50 亿美元，未来视商业进程追加最高 200 亿美元；Anthropic 承诺未来十年在 AWS 投入超 1000 亿美元**

- **算力资源：锁定最高 5 吉瓦亚马逊当前及下一代 Trainium 芯片算力（Trainium2/3/4），以及数千万颗 Graviton CPU 核心**

- **准入权益：AWS 客户可通过 Amazon Bedrock 第一时间访问 Anthropic 所有最新模型，包括 Claude Opus 4.8 的 Fast Mode**

- **谷歌 + 博通：5 吉瓦下一代 TPU 算力，解决未来算力渠道单一问题（2026年4月7日）**

- **合作规模：与谷歌和博通签署五年深度合作，总投入约 2000 亿美元**

- **算力资源：锁定 5 吉瓦下一代 TPU 算力，2026 年率先落地超 1GW 产能，2027 年全面上线**

- **准入权益：Google Cloud 客户可通过 Vertex AI 平台访问 Claude 全系列模型**

**笔者之前的详细分析参考：Anthropic年度ARR再创新高，迎来爆发性时刻，2026年的盈利近在眼前、服务多元化的子公司成立、以及算力渠道扩容后的商业闭环壁垒重构**[已存**https://www.acecamptech.com/article/detail/70562007**](https://www.acecamptech.com/article/detail/70562007)

**总结而言，近期Anthropic获得了市场足够的算力支撑，无论是渠道、供应链还是先有的算力规模等；另外再加上和不同云厂商签署的合作协议，如AWS、谷歌等，均可以在云端放置其最新的模型。所以，无论是底层的算力供应、还是应用层的模型分销都取得了不错的突破，加上Open AI Codex给到的压力，Anthropic需要做作出回应，所以出台了更新、更宽容的政策。在如此极大的利好背景之下，笔者认为市场每个月对于Anthropic的预期将都不一样**

# 4、总结

**1、Anthropic Cladue Opus 4.8的发布，无论是指标测试层面还是身边用户的深度使用和评价，均对比Claude Opus 4.7有所提升。不过价格问题依然是用户关注的重点，以及市场短暂的舆论并不会影响Anthropic的整体表现**

**2、此次迭代的时间很短，可以说是Anthropic有史以来最短的一次模型发布更新版本。笔者认为更多是Mythos在前期铺好了路，也就验证笔者之前谈到的，Anthropic有了技术实力（Mythos模型），后续的发布更多会进行技术迭代的控制，类似于之前Open AI的操作，避免技术被竞争对手破解**

**3、Open AI Codex的增长过猛，是Anthropic的留的禁止进入的空档给了一定的机会。当下，在Anthropic解决完算力问题之后，会加大针对竞争对手的打击，尽管其最新的套餐Agent SDK credits看起来挺实惠，但是依然有利可图，这里的利不仅仅是ARR的增长，更是对竞争对手的打压以及客户群体的维持，对于未来IPO的价值巨大**
