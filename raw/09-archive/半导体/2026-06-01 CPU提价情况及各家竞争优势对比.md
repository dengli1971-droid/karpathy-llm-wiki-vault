# CPU提价情况及各家竞争优势对比

**日期**: 2026-06-01 | **来源**: acecamptech.com | **公司**: AMD, AWS, Google, Intel, Meta, NVIDIA, OpenAI, Qualcomm, Samsung, TSMC, Tencent, 联想 | **标签**: 数据中心, 算力扩张, 供应链, 半导体 | **Ticker**: 005930 KS, 0700 HK, 2330 TT, 700 HK, AMD US, BC94 LN, GOOGL US, INTC US, META US, NVDA US, QCOM US, TSM US

## 正文

注：以下内容的数字部分，均来自个人搜集整理和行业专家调研，图表为笔者自制，可能存在不准确、不完整的情况，请自担风险






核心要点速览

1. 英伟达 ARM CPU 性能已超 AMD/Intel 最高端产品，但真正的竞争优势来自捆绑策略——只有选择英伟达 ARM CPU 的客户才能使用最新一代跨节点通信协议。

2. AMD 大客户折扣从前两年的 4-5 折收缩至当前 6-7 折，实际采购价格上涨约 40%-50%，远超市场传闻的 15% 涨幅。

3. 腾讯新增 CPU 采购已 100% 选择 AMD，停止采购英特尔；字节是唯一 AMD 占比 < 50% 的头部客户，因其更看重供应链稳定性。

4. CPU 供给极度紧张——正常 4-6 周交付期已延长至 20 周以上，头部客户提前支付定金锁定 Zen 6 产能，往年从未出现。

5. Zen 6 最高端型号算力密度已接近美国出口管制阈值，AMD 正在提前申报；若管制落地，将加速国内客户囤货并推动国产替代。

6. ARM 服务器 CPU 能效比优势获市场广泛认可，AMD 专家"无明显优势"的判断与主流认知存在偏差。

7. 云厂商自研 CPU 以自用为主，难以对外销售；IaaS 层客户大多指定 x86 架构，PaaS/SaaS 层优先使用自研 ARM，形成业务分层。

8. AMD 已在三星、格罗方德、英特尔 18A 完成流片作为台积电备选方案，但三星版 CPU 性能仅为台积电版的 6-7 折。







一、AMD市场份额
1.1 国内云厂商分客户份额拆分

专家信息： - 腾讯新增服务器CPU采购中AMD占比超80%，新增采购已全部选择AMD，不再采购英特尔 - 阿里新增采购AMD占比也高于英特尔，采购量增长已接近翻倍 - 字节是目前唯一英特尔采购量高于AMD的头部客户，AMD在字节CPU采购占比低于50%，原因是字节采购策略中TCO权重较低、更看重供应链稳定性 - 总需求规模排序：字节 > 阿里 > 腾讯。

全市场已知信息（对照）： - 公开研报仅披露AMD全球服务器CPU收入份额达41.3%（4Q25，Mercury Research数据），按出货量口径2025年AMD全球份额约28%-37% - 券商点评提及”2026M1 Intel、AMD服务器CPU涨价幅度在10%-15%，库存已全线售罄”，但未拆分到具体客户 - 增量判定：分客户维度的份额数据（腾讯80%+、字节<50%等）属于独家信息，公开渠道从未出现此精度的拆分




1.2 海外云厂商份额细节

专家信息： - AWS的x86服务器CPU采购中，AMD占比超过50% - AWS自研ARM芯片（Graviton）仅占其整体服务器芯片的约20%，剩余80%为x86架构 - 海外云厂商对AMD CPU的采纳进度比国内领先半年到一年 - 2025年海外增量市场AMD份额已超过50%

全市场已知信息（对照）： - 公开研报提及”AWS Graviton连续三年占AWS新增EC2算力的50%以上”，但会议给出的20%是存量口径，与公开信息的”新增”口径不同，两者并不矛盾但补充了存量渗透的真实水平 - 另一份专家交流提及”亚马逊Graviton ARM处理器用于AWS传统后台业务，GPU相关处理器仍以x86为主”，与会议信息一致 - 增量判定：AWS中AMD在x86部分超50%的份额、Graviton仅占整体20%的存量数据属于更高精度信息




1.3 海外客户分客户需求增速

专家信息： - Meta CPU需求增长超过1倍，主要因多个内部项目及real GW数据中心方案落地 - 谷歌 CPU需求增长约59% - 亚马逊 x86 CPU需求增长不高，增量需求优先由自研ARM芯片满足 - 整体数据中心CPU需求较2025年同比增长约50%

全市场已知信息（对照）： - AMD管理层在26Q1财报会上表示服务器CPU TAM将以每年超过35%的速度增长，到2030年将超过1200亿美元 - 公开研报预测2026年通用服务器出货增长从8%上修至12% - ARM CEO提及每GW数据中心CPU核心需求将增长至当前4倍 - 增量判定：分客户维度的需求增速（Meta超1倍、谷歌59%）属于独家数据，公开渠道仅有行业整体增速预测

 




二、AMD供需与价格
2.1 实际涨价幅度远超市场传闻

专家信息： - AMD不直接提高官方指导价，而是通过收缩折扣变相涨价 - 涨价上限约束：大客户折扣收缩至9-10折则属于行业临界点（触发ARM/RISC-V替代风险）

全市场已知信息（对照）： - 公开报道普遍引用”CPU涨价10%-15%” - 台媒报道AMD计划2026年Q2-Q3连续两次上调服务器CPU价格，累计涨幅预估16%-17% - 增量判定：实际涨价的幅度远超公开报道的10%-17%。公开报道仅反映了官方通知的价格调整，未捕捉到折扣收缩带来的隐性涨价




2.2 交货周期大幅延长

专家信息： - AMD承诺交货周期已从正常的4-6周（实际1-2周交付）延长至20周以上 - 头部客户已在Zen 6性能测试结果未出的情况下提前支付定金签订长期供货协议锁定产能，此情况往年从未出现 - 供货优先级：头部云厂商最高，其中阿里供货优先级最高；中小企业、政企客户缺货最明显 - 季度产能仅能通过封装等环节调整实现3%-10%的小幅增长

全市场已知信息（对照）： - 公开点评提及”AMD部分产品交付周期延长至8-10周”（2026年2月数据） - 公开报道提及”CPU平均交货期从1-2周延长至8-12周，部分场景长达6个月” - 增量判定：20周以上的交货周期远超公开信息中的8-12周口径，客户提前支付定金锁产能的行为属于首次披露




2.3 备选供应链详情

专家信息： - AMD已在三星、格罗方德、英特尔18A等工艺路线上完成流片，作为备选第二供应源 - 使用三星产能生产的CPU性能仅为台积电版本的6-7折 - IO die等非核心部件可使用格罗方德成熟工艺，但功耗会上升 - 仅在台积电产能极端紧张时才会启用备选方案，目前大概率不会出现

全市场已知信息（对照）： - 公开研报仅提及”AMD路线图高度绑定台积电先进制程”，未披露备选供应商的具体流片进展 - 增量判定：AMD已在三星完成流片、三星版CPU性能仅为台积电6-7折等信息

 




三、AMD产品路线图与技术
3.1 Zen 7制程规划

专家信息： - Zen 7不会采用2nm工艺，预计将使用台积电下一代A16制程

全市场已知信息（对照）： - 公开研报仅提及”Zen 7预计于2027-2028年左右发布，将首次搭载新矩阵引擎”，未披露具体制程选择 - 增量判定：Zen 7采用A16制程的信息属于未公开的产品路线图




3.2 性能领先幅度的精确量化

专家信息： - 当前AMD最高端192核CPU对比英特尔最高端256核CPU，整体性能领先30% - Zen 6（2nm）性能领先幅度将扩大到35% - 英特尔14A产品（预计2年后发布）即使完全按路线图交付，也仅会缩小差距，不会反超

全市场已知信息（对照）： - 公开研报提及”EPYC 9965性能提升29%、能效提升66%“，但这是与AMD自身上一代对比，非与英特尔直接对比 - AMD CEO在财报会上表示Venice”每插槽及每瓦性能大幅提升，相较ARM产品每插槽吞吐量提升逾2倍” - 增量判定：30%/35%的具体领先幅度数字以及对英特尔14A的前瞻性竞争评估属于内部竞争分析，公开渠道未有此精度




3.3 出口管制风险预警

专家信息： - Zen 6最高端型号的算力密度已接近美国出口管制阈值 - AMD正在提前向美国贸易管理局申报 - 下一代CPU如果美国不提升管制阈值，大概率会触发出口管制限制 - 该预期正在推动国内客户加速囤货

全市场已知信息（对照）： - 公开信息中”此前只有GPU达到美国出口管制的算力密度限制标准”，CPU接近管制阈值的信息未见公开报道 - 增量判定：CPU即将触及出口管制阈值、AMD主动申报、国内囤货加速等信息属于未公开的合规风险预警




3.4 CPU技术演进方向（逻辑差异）

专家信息： - 未来CPU将向中核、小核混合架构演进，不会全部堆大核，也不会完全走GPU的纯并行路线 - AMD正在预研CPU与内存之间的光互联技术，替代铜互联 - AMD正在预研存算一体技术，直接在内存侧实现运算 - 可能在CPU与内存之间增加HBM作为四级缓存 - 半导体行业3年内的产品规划在3年前就已确定，当前需求变化只影响3年后的产品

全市场已知信息（对照）： - 公开研报提及CPU核数提升趋势和Agent场景需求，但未涉及光互联、存算一体等前沿技术预研方向 - 增量判定：光互联预研、存算一体探索、HBM作为四级缓存等技术方向属于AMD内部研发路线的独家披露

 




四、ARM授权模式与收费
4.1 版税比例精确拆分（独家数据）

专家信息： - 标准core授权版税比例：10%-20% - 指令集/架构授权版税比例：2%-3%（因客户自行设计CPU，版税更低） - 前期授权费：标准core授权为千万级美金以上，最高可达上亿美金；架构授权更贵 - 两种模式长期总收益相对平衡：前期费高则后续版税低，反之亦然 - 英伟达芯片售价更高，虽版税比例更低，但版税绝对值可能超过亚马逊

全市场已知信息（对照）： - 公开研报仅笼统提及”ARM采用IP授权+版税模式”，测算假设”按照ARM CPU生态收入3.5%抽成” - Citi研报提及ARM面临”推高版税率的挑战”，但未给出具体比例 - 增量判定：10%-20%（标准core）和2%-3%（架构授权）的精确版税比例区间属于独家数据，公开研报中3.5%的假设仅为券商测算均值，未区分授权模式




4.2 英伟达与亚马逊合作模式差异（逻辑差异）

专家信息： - 亚马逊采用标准core授权，使用ARM最顶级核心产品，合作近10年，已迭代至Graviton 4，Graviton 5在研 - 英伟达采用指令集/架构授权，自研”奥林巴斯”架构，对ARM技术改造深度远高于亚马逊 - 两种模式的本质差异：亚马逊是”买成品改装”，英伟达是”买原材料自行设计”

全市场已知信息（对照）： - 公开研报提及”AWS Graviton、NVIDIA Grace/Vera均采用ARM生态”，但未区分二者授权模式的本质差异 - 增量判定：两种授权模式的具体差异及其对版税结构的影响属于增量信息

 




五、英伟达ARM CPU竞争力
5.1 新一代CPU性能参数超越x86（独家数据）

专家信息（ARM专家）： - 英伟达新CPU为88核双线程，总线程数176个 - 前端宽度10-wide，高于AMD的8-wide - 内存带宽达1.2TB，行业其他同类产品普遍为600GB-800GB - 性能已超过AMD和英特尔的高端CPU产品 - 较上一代Grace性能提升近50%

专家信息（AMD专家，对照视角）： - 英伟达ARM CPU本身性能弱于AMD x86 CPU - 英伟达的优势在于CPU+GPU+网络的捆绑方案，而非CPU单品性能 - NVLink C2C互联带宽1.8TB/s，是标准PCIe 6.0的2-3倍 - 英伟达的高带宽优势来自私有协议放弃生态兼容性，并非底层技术代差

高通公司视角：当前AI服务器市场中，x86架构占比约60%，ARM架构占40%。预计到2027年，两者比例可能达到5:5，而到2028年，ARM架构的占比或将反超至60%。 在ARM CPU成为主流的趋势下，竞争将围绕几个核心能力展开： 第一，单核效能，即性能与功耗的平衡。 第二，多核扩展能力，当前高端CPU已向200至400核发展。 第三，对先进制程的运用能力。

全市场已知信息（对照）： - 公开研报提及”NVLink5实现单GPU到GPU带宽1800GB/s”，但主要讨论GPU间互联，较少涉及CPU与GPU的互联 - 公开研报提及英伟达Vera CPU为”88核定制Olympus核心，搭配1.8TB/s NVLink-C2C” - 增量判定：两位专家对英伟达CPU竞争力的评估存在明显分歧——ARM专家认为性能已超越x86，AMD专家认为单品性能弱于AMD。这一分歧本身就是重要增量，揭示了评估维度的差异（单品CPU性能 vs 整体方案性能）。10-wide前端宽度、1.2TB内存带宽等具体参数在公开研报中较少出现




5.2 英伟达跨节点互联的捆绑策略（未公开战略）

专家信息（AMD专家）： - 英伟达的更大优势在于跨节点网络通信（基于收购Mellanox的IB技术），而非节点内互联 - 英伟达通过方案捆绑：仅对采用自家ARM CPU的客户开放最新版本的跨节点协议 - 选择标准x86 CPU的客户，英伟达仅允许使用上一代跨节点技术 - 英特尔未来部分CPU也会支持NVLink，但预计英伟达仅开放上一代版本

全市场已知信息（对照）： - 公开研报讨论NVLink主要聚焦节点内GPU互联，对跨节点捆绑策略的分析较少 - 有研报提及”ETL256将从NVLink scale-up过渡到混合架构”，但未涉及版本歧视策略 - 增量判定：英伟达对x86客户仅开放上一代跨节点协议的”版本歧视”策略属于独家信息，揭示了英伟达ARM CPU竞争力的真正来源

 




六、ARM市场渗透与生态
6.1 AWS ARM部署占比已超50%（更高精度）

专家信息： - 亚马逊数据中心内ARM架构CPU的部署比例已超过50% - 亚马逊是ARM在数据中心领域排名第一的合作伙伴

全市场已知信息（对照）： - 公开研报提及”Graviton CPU连续三年占AWS新增EC2算力的50%以上”，这是”新增”口径 - 另一份专家交流提及”亚马逊Graviton ARM处理器用于AWS传统后台业务” - AMD专家会议给出的数据是”AWS自研ARM芯片占整体服务器芯片的约20%” - 增量判定：两位专家给出的数据存在口径差异（ARM专家说”部署比例超50%“，AMD专家说”约20%“），可能分别对应”新增部署”和”存量占比”两个口径。这一口径差异本身就是重要增量信息




6.2 ARM能效比优势的精确量化（独家数据）

专家信息： - ARM精简指令集的能效比比x86高30%-50% - 同等功耗下可集成更多核心，并发处理能力更强 - 大型数据中心仅电费一年可节省上百亿美金

全市场已知信息（对照）： - 公开研报提及”ARM能效比优势，每瓦性能是x86的2倍”和”比X86省电约40%” - 增量判定：30%-50%的能效比优势区间与公开信息基本一致（40%处于该区间内），但”上百亿美金”的电费节省规模属于更具体的量化




6.3 谷歌自研CPU定位差异（逻辑差异）

专家信息： - 谷歌自研CPU主要服务于自身TPU数据中心体系和Youtube等媒体服务场景 - 谷歌技术层面仅落后亚马逊1-2年 - 谷歌与亚马逊产品定位差异化大，不存在绝对的追赶对齐需求

全市场已知信息（对照）： - 公开专家交流提及”谷歌自研ARM处理器仅用于谷歌云传统内部业务，占谷歌云20%多，不对外提供ARM云服务” - 增量判定：谷歌CPU服务于TPU数据中心和媒体业务的定位细节，以及与亚马逊仅1-2年技术差距的判断属于增量信息

 




七、ARM AGI CPU与新业务
